01 — Hardware

Custom Silicon.
Custom Boards.

We select and qualify every component. PCB designs are done in KiCad with attention to signal integrity, power distribution, thermal performance, and manufacturability. Nothing is left to defaults.

MCU Selection
STM32 — The Right Tool

The STM32 family from ST Microelectronics offers the best balance of performance, peripheral richness, community support, and long-term availability for embedded flight systems. The F405 provides Cortex-M4F performance for the FC; the F051 provides a lean, cost-effective Cortex-M0 for the ESC.

STM32F405 — FC STM32F051 — ESC ARM ecosystem Long-term supply
PCB Design
KiCad Schematics + Layout

All Aerolith boards are designed in KiCad — a professional, open-source EDA tool. This allows us to maintain complete version-controlled schematics and layouts, and enables full community inspection of the hardware under our OSHW license.

KiCad EDA 4-layer stack-up Impedance control DRC verified
Power Architecture
MOSFET Half-Bridge Power Stage

The LithDrive ESC power stage uses N-channel MOSFETs in a 3-phase half-bridge topology. Gate driver ICs provide bootstrap-based high-side switching with precisely timed dead-time insertion to prevent shoot-through conditions.

N-ch MOSFET Half-bridge Bootstrap gate drive Dead-time control
Component Philosophy
Qualify Everything. Trust Nothing.

Every component is evaluated against its datasheet limits before being committed to a design. Decoupling strategies, voltage derating, and thermal margins are analyzed at the schematic stage — not discovered after first power-on.

Derating policy Thermal modeling BOM qualification

Real-Time Architecture
Deterministic Execution

Flight-critical loops run in interrupt context or high-priority RTOS tasks with deterministic scheduling. Timer-based ISRs ensure sensor sampling and motor output happen at precise, predictable intervals — regardless of background processing load.

IRQ-driven loops FreeRTOS priorities No blocking I/O

Each peripheral — IMU, RC receiver, DSHOT motor output, telemetry UART — has a clean, tested driver layer. DMA is used extensively to offload CPU from communication overhead, keeping the control loop running at full speed.

03 — Control Systems

From Sensor to Motor.
In Microseconds.

The control pipeline processes raw IMU data, estimates attitude, computes PID corrections, and outputs motor commands — all within a tightly bounded execution window.

Attitude Estimation
Mahony / Complementary Filter

Accelerometer and gyroscope measurements are fused using a Mahony or complementary filter running in quaternion space — providing drift-corrected attitude estimates at rates exceeding 1 kHz.

Quaternion math >1 kHz update rate Drift correction
PID Control
Cascaded Rate + Attitude Loops

An outer attitude controller feeds setpoints into an inner rate controller. Configurable P/I/D terms, anti-windup, and derivative filtering provide stable and responsive flight. All parameters are tunable over the debug UART.

Cascaded PID Anti-windup Derivative filter UART tuning
Motor Mixing
Quadrotor X Mixing Matrix

PID outputs are converted to per-motor thrust commands using a standard quadrotor X mixing matrix, with output saturation and desaturation to prevent loss of control authority at high throttle.

X mixing matrix Output saturation Desaturation
BLDC Control
6-Step Trapezoidal (LithDrive)

Motor commands arrive at each LithDrive ESC as DSHOT frames. The ESC executes 6-step commutation independently, using back-EMF ZCD for timing, while returning RPM and current telemetry to LithCore on each cycle.

6-step Back-EMF ZCD DSHOT telemetry
